## Project Overview

### Purpose
Robofutures is an investment monitoring dashboard focused on the humanoid robotics value chain. It tracks:
- A 14-ticker universe across a 4-layer supply chain (Cyclical Semis, Japanese Actuators, System Integrators, ABF Substrates)
- Market performance relative to benchmarks (BOTZ robotics ETF, S&P 500 via SPY, NASDAQ 100 via QQQ)
- Four robotics market segments (humanoid, warehouse, collaborative, surgical)
- Leading indicators (patents, hiring, order books, policy, supply chain, earnings sentiment)
- Signal cockpit: cycle clock, momentum radar, action queue
- Watchlist management and dynamically derived alerts

### Target Deployment
- **Production:** GitHub Pages at `https://weltogeisto.github.io/Robofutures/`
- **Environment:** Static site with no backend services
- **Data:** Live quote/history data fetched from `public/data/` at runtime; hardcoded fallback values in `ALL_TICKERS`

### Current State
- **Status:** Active development
- **App.jsx:** 647 lines
- **index.css:** 746 lines (custom Linear-inspired design system)
- **Testing:** Formal test suite — `tests/run-all.mjs` runs `signalCockpit.test.mjs` and `stabilization.test.mjs`; invoked as `npm test` and as a CI step before build
- **TypeScript:** Not currently used (plain JavaScript/JSX)

`src/App.jsx` is the single working component (647 lines). It contains all inline data constants, state, and JSX. The files in `src/components/` are stubs not currently imported.

```

│   ├── ALL_TICKERS — 14 ticker objects (hardcoded fallback prices/metrics)
│   ├── SEG — 4 segment definitions
│   ├── MARKET — synthetic monthly chart data (DEAD CODE — never rendered)
│   ├── TIME_SCALES — timeframe selector options
│   └── ALL_DAILY — synthetically generated 3Y daily data (chart fallback)
│
├── Utility Components (lines 110–133)

│
└── App() Component (lines 135–647)
    ├── State & Effects (lines 136–218)
    │   ├── Fetch live data from public/data/ on mount
    │   ├── Merge live quotes into ALL_TICKERS
    │   └── Compute signalCockpit from timeframe-enriched tickers

    ├── Sidebar (lines 251–286) — navigation + watchlist
    ├── Header (lines 288–320) — data health badge + alert button
    │
    └── Tab Content (lines 337–628)
```

|----------|------|---------|

Derived values (`useMemo`):
- `mergedTickers` — ALL_TICKERS with live prices overridden from liveQuotes
- `timeframeTickers` — mergedTickers with `ch`/`rb` recalculated for selected timeframe
- `cockpit` — signal cockpit (cycle clock, momentum ranking, action queue)
- `alerts` — dynamically derived alerts (max 6)
- `chartDataRaw` — live history or ALL_DAILY synthetic fallback

### Signal Cockpit (`src/lib/signalCockpit.js`)

Key exports:
- `getDataHealth(quotesPayload, historyPayload)` — freshness/validity check
- `deriveDashboardAlerts({ dataHealth, tickers })` — momentum and data-health alerts
- `scoreTicker(ticker, data)` — earlyness/momentum/status per ticker
- `rankMomentumTickers(tickers)` — sorted by earlyness score
- `mapEcosystemLayers(tickers)` — bottleneck scoring by layer
- `classifyCycleStage(tickers)` — Phase 2/3/4 cycle clock
- `computeSignalCockpit(tickers)` — combines all of the above

## Working with This Codebase

### When Adding Features

1. All UI logic lives in `src/App.jsx` — read it first
2. Data constants are at the top (lines 1–88); add new ticker/segment data there
3. New tab: add a `NAV` entry + a `{tab === 'newtab' && ...}` block in the JSX
4. Use CSS variables for styling; avoid hardcoding colors
5. Run `npm test` before committing

### When Modifying Ticker Data

Ticker metadata lives in `ALL_TICKERS` (App.jsx lines 14–29). Live prices are overlaid from `public/data/quotes.json` at runtime. The `h` (52w high) and `w` (52w low) fields are **not** automatically updated from live quotes — update them manually when they become stale.

To add a ticker to the live data pipeline, also add it to `TICKERS` in `scripts/update_data.py`.

### Unused Code to Be Aware Of

- `const MARKET` in App.jsx: synthetic monthly data, never rendered — dead code
- `src/components/Sidebar.jsx`: bugs (undefined `Bot` and `unread`), not imported
- `src/components/OverviewTab.jsx`: empty, not imported
- `src/data/*.json`: generated by `scripts/update-data.mjs` (legacy JS fetcher) but not imported by App.jsx

## Data Architecture

### Two-Tier Data System

**Tier 1 — Hardcoded fallback (`ALL_TICKERS` in App.jsx):**
14 ticker objects with: price `p`, 52w high `h`, 52w low `w`, % change `ch`, rebound `rb`, robotics exposure `ex`, tier `ti`, sector `se`. Used as fallback when live data unavailable; values are stale by definition.

**Tier 2 — Live data (`public/data/`):**
- `quotes.json` — latest price, daily change%, prev close, 52w high/low, market cap, volume per ticker
- `history.json` — 3Y daily closes for BOTZ, SPY, QQQ, plus per-ticker close arrays

Generated by `scripts/update_data.py` (Python, `yfinance`). Intended to run daily via cron.

### Live Data Merge Logic

```javascript
// Price + daily change override (mergedTickers)
merged[sym] = { ...ALL_TICKERS[sym], p: q.price, ch: q.change_pct, live: true };

// Timeframe-specific ch/rb recalculation (timeframeTickers, when per-ticker history available)
ch = (latest - prior) / prior * 100;
rb = (latest - periodLow) / (periodHigh - periodLow) * 100;
```

When `liveHistory.tickers` per-ticker data is unavailable, `ch` falls back to the daily `change_pct` from quotes — misleading for the default 6M timeframe display.

### Chart Data Sources

| Series | Live Source | Fallback |
|--------|------------|---------|
| Robotics Index | BOTZ ETF daily closes (`history.json`) | Synthetic random walk (`ALL_DAILY`) |
| S&P 500 | SPY daily closes (`history.json`) | Synthetic random walk |
| NASDAQ | QQQ daily closes (`history.json`) | Synthetic random walk |

All series are rebased to 100 at the start of the selected timeframe.

### Segment Definitions (SEG in App.jsx)

| ID | Name | Color | Momentum |
|----|------|-------|----------|
| humanoid | Humanoid | #8b5cf6 | 97% (hardcoded) |
| warehouse | Warehouse | #3b82f6 | 93% (hardcoded) |
| cobot | Collaborative | #10b981 | 89% (hardcoded) |
| surgical | Surgical | #ec4899 | 86% (hardcoded) |

Note: The `g` (growth rate) field in SEG is defined but not rendered in the current UI.

### Alert System

Alerts are derived dynamically via `useMemo` — not stored as state. `deriveDashboardAlerts()` generates:
- Data-health alerts (stale/missing data)
- High-momentum alerts (`ch ≥ 75%` on selected timeframe)
- Crowding alerts (`rb ≥ 98%`)

Maximum 6 alerts, ordered by priority.

### Data Health States

| Status | Label | Condition |
|--------|-------|-----------|
| `fresh` | LIVE | Data loaded, < 72h old, no failed symbols |
| `degraded` | DEGRADED | Some symbols failed to fetch |
| `stale` | STALE | Data > 72h old |
| `missing` | NO DATA | No payload loaded |

## Important Considerations

### Dead Code / Known Issues

- `const MARKET` in App.jsx: synthetic monthly data, defined but never rendered
- `src/components/Sidebar.jsx`: references undefined `Bot` and `unread`; not imported by App.jsx
- `src/components/OverviewTab.jsx`: empty file, not imported
- `src/data/*.json`: written by `update-data.mjs` but not read by App.jsx; the Python `update_data.py` is the active data pipeline

### 52w High/Low Staleness

The `h` and `w` fields in `ALL_TICKERS` are not automatically updated from live `quotes.json` even though that file contains current `high_52w` and `low_52w` values. They serve as fallback inputs to `rb` calculations only. When live ticker history is loaded, `rb` is recalculated from actual period prices.

### Performance

Good for current dataset size (14 tickers). No memoization issues.

### Security

Static site, no user auth, no backend. No XSS vectors (React escapes by default). No API keys required (`yfinance` uses public Yahoo Finance endpoints).
